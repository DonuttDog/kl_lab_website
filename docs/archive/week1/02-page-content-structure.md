## 全站顶栏（布局组件，参考 Pixso「分区 + 控件类型」）

全站多数页面共用 **MainLayout**；**P-04 测评作答** 建议使用 **无顶栏布局**，避免干扰（与 [01-page-inventory.md](01-page-inventory.md) 一致）。以下为顶栏**功能结构**（非视觉规范）。

| 区域 | 模块 | 字段 / 内容 | 用户动作 | 跳转 |
|------|------|-------------|----------|------|
| 左 | Logo | 平台名称 / Logo | 点击 | `/` |
| 左 | 导航 · 测评 | 文案「心理测评」或「测评」；带 **下拉** | 悬停或点击展开 | 下拉内：热门/分类预览链至 `/assessments` 或 `/assessments/:id`（Mock） |
| 左 | 导航 · 商城 | 文案「商城」（对应需求中的 **课程** 列表，见用语表） | 悬停或点击展开 | 下拉内：推荐课程预览链至 `/courses` 或 `/courses/:id` |
| 右 | 账户 | 「登录」 | 点击 | `/login`（前端 mock 登录页） |
| 右 | 主 CTA | 「开始测评」等 | 点击 | `/assessments` 或某套默认测评 `/assessments/:id`（产品定） |

说明：与 Pixso 类似，**左侧**为多分区导航 + **可下拉预览**；**右侧**为 **登录 + 强 CTA**。Week2 已完成视觉统一改造与 Tailwind 渐进落地，见 [../week2/06-styling-tech-decision.md](../week2/06-styling-tech-decision.md)。

**i18n**：顶栏与各页模块的可见文案对应 [src/locales/zh-CN.json](../../src/locales/zh-CN.json) 中的 `nav.*`、`header.*`、`page.*`、`mock.*` 等 key；组件内不出现硬编码中文句子（Mock 卡片标题亦用 `mock.*` key）。

## P-01 首页

| 模块 | 字段 / 内容 | 用户动作 | 跳转 |
|------|-------------|----------|------|
| 顶栏 | 见上文「全站顶栏」（含 Logo → `/`） | — | — |
| 首屏 / 平台介绍 | 标题、副文案：平台定位与核心理念 | 无 | — |
| 功能入口 | 两大入口：测评、课程 | 点击「测评」 | `/assessments` |
| 功能入口 | 同上 | 点击「课程」 | `/courses` |
| 产品特点 | 2～4 条：专业性、体验等短句 | 无 | — |
| 页脚（可选） | 版权、备案占位 | 无 | — |

**核心任务**：建立第一印象，并在一屏内可进入测评或课程一级页（符合「核心功能在一级可直达」）。

## P-02 测评列表

| 模块 | 字段 / 内容 | 用户动作 | 跳转 |
|------|-------------|----------|------|
| 页标题区 | 标题「心理测评」、说明文案（解释筛选与标签含义） | 无 | — |
| 搜索 | 按测评名称关键词（字典中的标题） | 输入关键词 | 前端过滤 Mock |
| 筛选 | **三个下拉**，独立维度：**内容主题**、**适用人群**、**使用场景**；各有「全部」与带说明选项；多条件 **AND** | 更改下拉 | 刷新列表（Mock） |
| 测评卡片列表 | 名称、**三个归类标签**（与下拉语义一致）、简介、适用人群、预计时长 | 点击「查看详情」 | `/assessments/:id` |
| 测评卡片列表 | 同上 | 点击「开始测评」 | `/assessments/:id/answer` |
| 空状态 | 无数据时提示（建议放宽筛选/清空搜索） | 可选：去首页 | `/` |

**核心任务**：在可理解的筛选与搜索下选择一套测评，进入详情或开始作答。

**说明**：已弃用「主题 / 对象 / 用途」三 Tab 的模糊切换，改为「搜索 + 三下拉 + 卡片标签」，避免用户不清楚每个按钮代表的维度。实现见 [AssessmentListPage.tsx](../../src/pages/AssessmentListPage.tsx)。

## P-03 测评详情

| 模块 | 字段 / 内容 | 用户动作 | 跳转 |
|------|-------------|----------|------|
| 面包屑（可选） | 首页 > 测评 > 当前名称 | 点击「测评」 | `/assessments` |
| 标题区 | 测评名称、预计时长、适用人群 | 无 | — |
| 详细介绍 | 测评目的、说明、注意事项（富文本占位） | 无 | — |
| 主操作 | 主按钮「开始测评」 | 点击 | `/assessments/:id/answer` |
| 次操作（可选） | 「返回列表」 | 点击 | `/assessments` |

**核心任务**：读完信息后明确进入作答；路径单一，避免与列表重复堆信息。

## P-04 测评作答

| 模块 | 字段 / 内容 | 用户动作 | 跳转 |
|------|-------------|----------|------|
| 极简顶栏 | 测评名称、关闭/退出（需二次确认占位） | 确认退出 | `/assessments/:id` 或 `/assessments` |
| 进度 | 当前题号 / 总题数或进度条 | 无 | — |
| 题目区 | 题干、题型（单选/多选/量表等占位） | 选择选项 | 本地状态 |
| 导航 | 「上一题」「下一题」；末题显示「提交」 | 下一题 / 提交 | 提交成功占位页或 Toast（后端未就绪时 Mock） |

**核心任务**：单页聚焦作答，减少导航与营销信息（符合文档「简洁、聚焦、少干扰」）。

## P-05 课程列表

| 模块 | 字段 / 内容 | 用户动作 | 跳转 |
|------|-------------|----------|------|
| 页标题区 | 标题「课程」、一句说明 | 无 | — |
| 筛选 | 维度占位：如适用对象、主题（具体枚举待内容方定） | 变更筛选条件 | 刷新列表（Mock） |

**核心任务**：浏览与筛选后进入课程详情。

## P-06 课程详情

| 模块 | 字段 / 内容 | 用户动作 | 跳转 |
|------|-------------|----------|------|
| 面包屑（可选） | 首页 > 课程 > 当前名称 | 点击「课程」 | `/courses` |
| 头图区 | 封面大图、课程名称 | 无 | — |
| 元信息 | 适用对象、时长/节数等（占位字段） | 无 | — |
| 课程介绍 | 完整内容、学习价值、大纲列表占位 | 无 | — |
| 主操作（占位） | 「立即学习」或「咨询报名」等，视商业规则后续定 | 点击 | 外链或 `#` / Toast 占位 |

**核心任务**：完整呈现课程内容与学习价值；第一周可无真实「学习页」，按钮占位即可。

## 展示顺序小结

1. **首页**：介绍 → 入口 → 特点。  
2. **列表页**：标题 → 筛选 → 卡片流。  
3. **测评详情**：标题与元信息 → 正文 → 主 CTA。  
4. **作答**：顶栏与进度 → 题目 → 导航。  
5. **课程详情**：封面与标题 → 元信息 → 正文 → CTA。
