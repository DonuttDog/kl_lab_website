# 04｜测评页面 UI-only 设计规范

## 目标

仅定义测评页面视觉与交互形态，不实现具体某一测评内容逻辑；确保页面可用于后续任意问卷内容填充。

## 覆盖页面

- 测评列表页（入口与筛选）
- 测评详情页（说明与开始）
- 测评作答页（题目展示、选项交互、提交前后状态）

## 测评列表页（UI）

- 顶部：页面标题 + 简短说明（说明筛选逻辑）。
- 控制区：搜索框 + 三下拉（内容主题/适用人群/使用场景）。
- 结果区：统一卡片布局（标题、标签、简介、时长、操作按钮）。
- 空状态：图标/占位文案 + `清空筛选` 按钮。

## 测评详情页（UI）

- 面包屑：返回列表路径可见。
- 头部信息：名称、预计时长、适用人群。
- 正文：目的、说明、注意事项分段展示。
- 行动区：主按钮“开始测评”，次按钮“返回列表”。

## 测评作答页（UI，参考图3）

### 页面结构

`极简顶栏 -> 题目分组块 -> 每题选项 -> 底部操作区`

### 题目块样式

- 每题使用白底容器卡片，题号与题干固定在卡片上方。
- 题与题之间保持统一大间距，避免视觉黏连。

### 题型 1：单选题

- 选项前置圆形单选控件。
- 点击整行可选中，非仅点击圆点。

### 题型 2：X 分量表（Likert，图3为 7 分示例）

- 左右两端显示语义锚点（如“非常消极”到“非常积极”）。
- 中间 `1~X` 按钮等宽排列（X >= 2），选中态高亮。
- 未选择前不默认值，减少误导。

## 状态规范

- 未作答：保持中性边框，无警示色。
- 已作答：选中项高亮，非选中项维持中性。
- 提交前校验：未答题显示就近提示（例如“请完成本题”）。
- 提交后：成功占位页或提示条，不进入真实报告页（Week2 仅 UI）。

## 文案规范

- 题干保持客观中性，避免带倾向性词汇。
- 操作文案统一：`上一题`、`下一题`、`提交`。
- 错误提示统一句式：`请先完成当前题目`。

## 可访问性要点

- 选项需支持键盘选择与焦点可见。
- 量表题每个数字按钮需有可读标签（如 `评分 1`）。
- 文本对比度满足可读性要求，避免浅灰文本过多。

## 为什么有助于三大目标

- 用户友好：题目结构清晰，交互反馈及时。
- 降低理解成本：单选与量表题都遵循统一视觉模式。
- 建立信任感：界面克制、稳定，不以花哨样式干扰答题。
