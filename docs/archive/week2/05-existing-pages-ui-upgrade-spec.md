# 05｜已有页面 UI 升级规范（首页/测评/课程）

## 目标

在不改变 Week1 页面职责的前提下，统一已有页面视觉与交互质量，减少各页“像不同产品”的割裂感。

## 升级范围

- 首页：信息层级、入口卡片、按钮一致性。
- 测评：列表筛选区、卡片、详情排版、作答控件一致性。
- 课程：列表卡片和详情信息区的统一组件化表达。

## 页面级升级规则

### 首页

- Hero 文案长度控制：主标题 1 行，副标题 1~2 行。
- 两大入口卡片采用同尺寸同结构（图标、标题、一句说明、按钮）。
- 特点区改为 3~4 条短句卡片，视觉权重低于功能入口。

### 测评列表

- 搜索与三下拉筛选放入同一“控制条面板”，统一高度与间距。
- 卡片内信息顺序固定：标题 -> 三维标签 -> 简介 -> 元信息 -> 行为按钮。
- `开始测评` 固定主按钮样式，`查看详情` 固定次按钮样式。

### 测评详情

- 标题区增加“预计时长/适用人群”元信息行，和课程详情格式一致。
- 说明正文使用统一段落间距，禁用彩色强调块滥用。
- 主 CTA 置顶可见区域内，避免用户滚动后找不到操作。

### 测评作答

- 题干区与选项区固定最大宽度，提高可读性。
- X 分量表类题目使用等宽选项按钮（X >= 2），数字居中。
- 翻页按钮固定底部区域，位置不随题目高度变化。

### 课程列表

- 课程卡片统一为“封面 + 标题 + 简介 + 适用对象”四段结构。
- 封面比例统一（建议 16:9），避免列表抖动。

### 课程详情

- 头图区与正文信息区分层明确，防止信息堆在首屏。
- 使用“你将学到什么 / 课程大纲 / 适用对象”固定顺序。
- 主按钮文案统一（例如“立即学习”），避免同义词混用。

## 交互一致性清单

- 按钮：高度、圆角、字号三件套统一。
- 表单：输入框、下拉、错误提示统一。
- 卡片：边框、阴影、悬浮反馈统一。
- 标签：统一颜色语义，不按页面随意换色。
- 空状态：统一插画风格（或统一无插画文本风格）。

## 不改动项（避免重复 week1）

- 不修改页面路由与流程定义。
- 不新增业务逻辑与后端联动。
- 不重写 Week1 的功能说明文档。

## 为什么有助于三大目标

- 用户友好：主操作和反馈规律一致，降低误触和迷失。
- 降低理解成本：跨页面视觉语言一致，用户迁移成本低。
- 建立信任感：视觉质量稳定，体现产品成熟度和专业度。
