# 02｜全页面布局与导航统一规范

## 目标

为首页、测评、课程、“我的”提供统一布局骨架和导航行为，让用户在任意页面都能快速判断“我在哪里、下一步去哪里”。

## 全站布局骨架

### 桌面端（>= 1200）

- 页面容器：`max-width: 1200px`，左右内边距 `24px`。
- 主内容区节奏：区块纵向间距采用 `24/32px` 两档。
- 卡片系统：统一白底、`1px` 边框、`radius-md`，避免特殊形状。

### 平板（768 - 1199）

- 容器宽度改为自适应，左右内边距 `16px`。
- 双列布局可降为 1~2 列混排，保证信息不拥挤。

### 手机（< 768）

- 单列布局，卡片纵向堆叠。
- 顶栏保留核心入口（Logo、主导航、头像菜单）。

## 全站导航结构

- 一级：`首页`、`心理测评`、`课程`、`我的(头像入口)`。
- 二级：由对应页面内部导航或下拉菜单承接。
- 回退路径：`Logo -> 首页` 固定可用。

## 顶栏规范（含图1参考）

### 结构

- 左侧：Logo + 主导航（测评、课程）。
- 右侧：通知（可选）+ 头像（必有）+ 主操作按钮（可选）。

### 头像菜单信息架构

1. 用户中心（进入“我的”）
2. 更新日志
3. 访问官网
4. 切换深色模式
5. 退出登录（当前为 mock 占位）

### 交互规则

- 点击头像打开菜单；再次点击或点击外部区域关闭。
- `Esc` 关闭菜单；上下键支持菜单项聚焦切换（可访问性）。
- 菜单项高度与左右内边距统一，不出现“某项更大/更小”。
- `退出登录` 使用危险色文案，但仅触发 mock 提示，不清理真实会话。

## 信息层级规则（降低理解成本）

- 每页仅一个 H1。
- 操作优先级固定：主按钮 > 次按钮 > 文本链接。
- 长页面必须按“标题块 -> 操作块 -> 内容块”分段，分段标题风格一致。

## 页面级骨架映射

### 首页

`顶栏 -> Hero -> 两大入口 -> 特点区 -> 页脚`

### 测评列表

`顶栏 -> 标题与说明 -> 搜索/筛选 -> 卡片列表 -> 分页或空状态`

### 测评详情

`顶栏 -> 面包屑 -> 标题与元信息 -> 正文 -> 主 CTA`

### 测评作答

`极简顶栏 -> 进度 -> 题目块 -> 上下题/提交`

### 课程列表

`顶栏 -> 标题 -> 筛选 -> 课程卡片 -> 空状态`

### 课程详情

`顶栏 -> 面包屑 -> 封面与标题 -> 介绍与大纲 -> CTA`

### “我的”页面

`顶栏 -> 页面标题 -> 左侧导航 + 右侧详情面板`

## 状态规范

- 空状态：说明原因 + 一条可执行动作（如“清空筛选”）。
- 错误状态：描述问题 + 重试按钮。
- 加载状态：骨架屏或占位块，避免布局跳动。

## 为什么有助于三大目标

- 用户友好：全站入口和回退路径固定，学习一次即可复用。
- 降低理解成本：页面骨架统一，用户能预判信息位置。
- 建立信任感：结构稳定、反馈一致，减少“随机感”和不确定性。
