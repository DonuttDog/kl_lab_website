# 07｜交付清单与前端交接规范

## 目标

确保 UI/UX 方案可被稳定接入开发，减少“设计稿完整但实现偏差大”的问题。

## 设计交付清单（Pixso）

- [ ] 页面画板完整：首页、测评列表/详情/作答、课程列表/详情、“我的”。
- [ ] 每页至少包含 `default` 状态；关键交互含 `hover/focus/disabled`。
- [ ] 组件已抽象到组件库页（按钮、输入、卡片、导航、量表、字段行）。
- [ ] 标注齐全：尺寸、间距、字号、颜色、圆角、阴影。
- [ ] 颜色与间距使用变量命名，不使用“临时色值”。
- [ ] 头像菜单（图1）、用户中心（图2）、量表题（图3）已单独标注。

## 前端接入前核对

## 验收标准（UI/UX）

- 一致性：同类组件在所有页面外观与交互一致。
- 可理解性：首次访问用户可在 3 秒内识别当前页面主任务。
- 可靠感：无明显跳色、过度动画、对齐错乱、层级混乱。
- 可扩展性：新增页面可复用现有组件与 token。

## 风险与规避

- 风险：设计稿命名不统一，导致开发映射困难。  
  规避：强制执行组件命名规范与变量命名规范。
- 风险：页面样式先写后统一，返工大。  
  规避：先落 token 与基础组件，再进入页面开发。
